# 風險登記簿（Risk Register）

專案級風險的整合視圖（逐票 severity 之外）。可能性 × 影響 → 風險等級 → 緩解 → 殘餘。

| ID | 風險描述 | 可能性(H/M/L) | 影響(H/M/L) | 等級 | 緩解措施 | 負責人 | 殘餘風險 | 狀態 |
|---|---|---|---|---|---|---|---|---|
| R1 | <外部依賴未到位> | M | H | 高 | <備案/排程> | | <可控?> | open |
| R2 | <無退場機制> | L | H | 中 | <prod 監控/smoke> | | | open |
| R3 | <資安未掃> | M | H | 高 | <ZAP/SAST 上線前> | | | open |

## 用法
- 等級 = 可能性 × 影響（高×高=最高優先）。
- 每個高風險 → 至少一個緩解 + 對應測試/檢查。
- 殘餘風險（緩解後仍存）要明確接受並記錄（誰拍板）。
- 連到 RTM/report：風險 → 對應測試覆蓋 → 是否驗證。
