# 測試技術與素質（從實作淬煉，通用）

## 技術（reusable methods）
1. **黑箱偵察**：無原碼時，從部署的 minified bundle 反推 端點 / 版本 / 對照表 / 頁面清單。
2. **人工關卡半自動**：captcha 截圖放大(nearest-neighbor)視覺辨識 + 手輸；持久 session/profile 重用免重登；標清楚哪些是 bypass。
3. **網路/DOM 層檢視**：攔截 API 回應看「真錯誤」(非 UI 文字)；斷言元素**狀態**(readonly/disabled/value/enabled)，不是只看可見字。
4. **故障隔離**：我方 vs 依賴方 —— 直打依賴(contract)定位誰錯；狀態機**逐態探測**定位卡在哪一步。
5. **加密產出驗證**：還原/推導密碼、驗加密參數(版本/位元)、解密查內容對不對人。
6. **並發重現**：同時發多請求，重現 race / 佇列堆積 / 鎖等問題（手動或 k6）。
7. **部署偵測**：bundle hash / 版本變動 = 重新部署，修正可能已上線，舊判定要重驗。
8. **跨層三角驗證**：data(API實值) × display(對照/UI) × code(寫入邏輯)。驗「寫入端」，別只看顯示就下結論。

## 素質（mindset，決定精準度）
- **反例優先**：先想「怎麼弄壞它」(漏填/越權/回退/並發)，再想正例。
- **質疑自己**：別用 舊資料 / 抽樣 / 「程式碼有那段」就下結論。
- **誠實標不確定**：無法驗就寫「無法判定 + 原因 + unblock」，不給假 PASS。
- **證據驅動**：每個判定配一個可重現 artifact。
- **找不到 ≠ 沒有**：換 2-3 個角度(關鍵字/端點/來源)再下「沒做」。
- **料的新鮮度**：驗修正只用修正後的乾淨案例。
